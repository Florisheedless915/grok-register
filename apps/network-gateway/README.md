# network-gateway

前置网络出口模块。

它不和注册脚本强耦合，职责是给浏览器和邮箱请求提供稳定出口，典型实现是本机或同机容器里的 WARP / 代理桥接。

建议职责：

- 提供浏览器侧出口，例如 `http://127.0.0.1:18118`
- 提供普通 HTTP 请求出口，必要时和浏览器出口分离
- 在任务启动前做健康检查
- 暴露可观测信息，例如当前出口 IP、连通性、被风控情况

在当前项目里，它通过两个配置项接入注册流程：

- `browser_proxy`：浏览器访问 `x.ai` 时走的出口
- `proxy`：邮箱 API 等普通请求走的出口

如果你已经在宿主机上跑好了 WARP 桥接，这个模块不需要额外改代码，只要把代理地址填进配置即可。
